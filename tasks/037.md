# 037 — Database Schema for Platform

**Status**: pending
**Phase**: 5 — Platform
**Dependencies**: 036
**Estimated effort**: medium (2-8h)

## Context

Current SQLite schema only stores conversations. Platform needs: agent configs, connections (OAuth tokens), schedules, knowledge sources, and audit logs all in the DB instead of code/env vars.

## Deliverables

### New tables

```sql
-- Agent configurations (replaces hardcoded CtoAgent)
CREATE TABLE agents (
  id TEXT PRIMARY KEY,
  name TEXT NOT NULL,
  model TEXT NOT NULL DEFAULT 'claude-sonnet-4-6',
  system_prompt TEXT NOT NULL,
  mcp_servers TEXT NOT NULL DEFAULT '[]',     -- JSON array of connection IDs
  knowledge_sources TEXT NOT NULL DEFAULT '[]', -- JSON array of source IDs
  permissions TEXT NOT NULL DEFAULT '{}',       -- JSON: canWrite, requiresApproval, etc.
  max_turns INTEGER NOT NULL DEFAULT 15,
  enabled INTEGER NOT NULL DEFAULT 1,
  created_at TEXT NOT NULL DEFAULT (datetime('now')),
  updated_at TEXT NOT NULL DEFAULT (datetime('now'))
);

-- OAuth connections (replaces .env credentials)
CREATE TABLE connections (
  id TEXT PRIMARY KEY,
  name TEXT NOT NULL,           -- "Jira", "GitHub", "Google Calendar"
  type TEXT NOT NULL,           -- "jira", "github", "google", "slack", "custom_mcp"
  status TEXT NOT NULL DEFAULT 'disconnected', -- connected, disconnected, error
  config TEXT NOT NULL DEFAULT '{}',           -- JSON: base_url, scopes, etc.
  credentials TEXT,                             -- JSON: encrypted OAuth tokens
  mcp_server_url TEXT,                         -- URL for remote MCP server
  last_sync_at TEXT,
  error_message TEXT,
  created_at TEXT NOT NULL DEFAULT (datetime('now')),
  updated_at TEXT NOT NULL DEFAULT (datetime('now'))
);

-- Schedules (replaces hardcoded cron jobs)
CREATE TABLE schedules (
  id TEXT PRIMARY KEY,
  name TEXT NOT NULL,
  cron_expression TEXT NOT NULL,
  agent_id TEXT NOT NULL REFERENCES agents(id),
  prompt TEXT NOT NULL,          -- what to ask the agent
  notification_channels TEXT NOT NULL DEFAULT '["telegram"]', -- JSON array
  enabled INTEGER NOT NULL DEFAULT 1,
  last_run_at TEXT,
  last_result TEXT,
  created_at TEXT NOT NULL DEFAULT (datetime('now')),
  updated_at TEXT NOT NULL DEFAULT (datetime('now'))
);

-- Knowledge sources (replaces hardcoded vault path)
CREATE TABLE knowledge_sources (
  id TEXT PRIMARY KEY,
  name TEXT NOT NULL,
  type TEXT NOT NULL,            -- "obsidian", "notion", "confluence", "filesystem", "s3"
  config TEXT NOT NULL DEFAULT '{}', -- JSON: path, api_key, workspace_id, etc.
  status TEXT NOT NULL DEFAULT 'disconnected',
  last_sync_at TEXT,
  doc_count INTEGER DEFAULT 0,
  error_message TEXT,
  created_at TEXT NOT NULL DEFAULT (datetime('now')),
  updated_at TEXT NOT NULL DEFAULT (datetime('now'))
);

-- Audit log (every agent action)
CREATE TABLE audit_log (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  agent_id TEXT,
  action TEXT NOT NULL,          -- "tool_call", "mcp_request", "knowledge_read", "knowledge_write"
  details TEXT NOT NULL DEFAULT '{}', -- JSON: tool name, args, result summary
  source TEXT NOT NULL,          -- "telegram", "web", "cron"
  created_at TEXT NOT NULL DEFAULT (datetime('now'))
);
```

### Concurrency: Atomic task checkout (learned from Paperclip)

```sql
-- Task execution locks (prevents double-work in batch/multi-agent)
CREATE TABLE task_checkouts (
  id TEXT PRIMARY KEY,
  task_ref TEXT NOT NULL,          -- external ref: Jira issue key, schedule ID, etc.
  agent_id TEXT NOT NULL REFERENCES agents(id),
  run_id TEXT NOT NULL,            -- unique per execution run
  session_id TEXT,                 -- Claude session ID for resume across runs
  checked_out_at TEXT NOT NULL DEFAULT (datetime('now')),
  released_at TEXT,
  status TEXT NOT NULL DEFAULT 'active' -- active, completed, cancelled, timed_out
);

CREATE UNIQUE INDEX idx_task_checkout_active
  ON task_checkouts(task_ref) WHERE status = 'active';
```

Checkout is atomic via the unique index — second agent trying to checkout the same task gets a constraint violation (equivalent to Paperclip's 409 Conflict). Session ID enables resuming Claude sessions across heartbeats.

### Budget tracking (learned from Paperclip)

```sql
-- Cost tracking per agent per period
CREATE TABLE cost_events (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  agent_id TEXT NOT NULL REFERENCES agents(id),
  run_id TEXT,
  input_tokens INTEGER NOT NULL DEFAULT 0,
  output_tokens INTEGER NOT NULL DEFAULT 0,
  cost_cents INTEGER NOT NULL DEFAULT 0,  -- in USD cents
  created_at TEXT NOT NULL DEFAULT (datetime('now'))
);

-- Budget policies
CREATE TABLE budget_policies (
  id TEXT PRIMARY KEY,
  scope TEXT NOT NULL,             -- 'agent', 'global'
  scope_id TEXT,                   -- agent ID if scope=agent, null if global
  monthly_limit_cents INTEGER NOT NULL,
  soft_threshold_pct INTEGER NOT NULL DEFAULT 80,
  hard_threshold_pct INTEGER NOT NULL DEFAULT 100,
  action_on_hard TEXT NOT NULL DEFAULT 'pause', -- pause, notify_only
  created_at TEXT NOT NULL DEFAULT (datetime('now'))
);
```

When hard threshold is hit: agent is paused, active checkouts are cancelled, a human approval is required to resume. Prevents runaway API costs.

### Approval queue (learned from Paperclip)

```sql
-- Human approval queue for sensitive actions
CREATE TABLE approvals (
  id TEXT PRIMARY KEY,
  agent_id TEXT NOT NULL REFERENCES agents(id),
  type TEXT NOT NULL,              -- 'budget_override', 'sensitive_action', 'deployment'
  status TEXT NOT NULL DEFAULT 'pending', -- pending, approved, rejected
  context TEXT NOT NULL DEFAULT '{}',     -- JSON: what the agent wants to do
  resolved_by TEXT,                -- user who resolved
  resolved_at TEXT,
  created_at TEXT NOT NULL DEFAULT (datetime('now'))
);
```
