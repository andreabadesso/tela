# 042 — Multi-Agent Orchestrator

**Status**: pending
**Phase**: 5 — Platform
**Dependencies**: 039, 040, 041
**Estimated effort**: large (8-20h)

## Context

Currently one monolithic agent handles everything. This task builds an orchestrator that routes requests to the right specialized agent, can spawn sub-agents for complex queries, and manages multi-agent conversations.

### Lessons from Paperclip analysis

Paperclip uses a heartbeat model where agents sleep and wake on events. All agent communication is async via issues/comments — no direct messaging. Key patterns to adopt:

1. **Atomic task checkout** — prevents double-work (our schema already has this from 037)
2. **Session persistence** — Claude sessions resume across runs via `--resume sessionId`
3. **Budget-gated execution** — agent paused when budget exceeded, human approves to resume

Key patterns we do **differently** (and better):

1. **Claude Agent SDK instead of CLI subprocess** — native streaming, MCP built-in, sub-agents supported in-process
2. **`ask_agent` as MCP tool** — agents call each other directly in the reasoning loop, no HTTP roundtrip
3. **Synchronous chat + async batch** — Paperclip is async-only; we support both real-time chat and overnight batch

## Deliverables

### Orchestrator class

```typescript
class Orchestrator {
  // Real-time: route a chat message to the best agent
  async chat(input: AgentInput): Promise<AgentOutput> {
    const agent = await this.resolveAgent(input);
    return agent.process(input);
  }

  // Batch: assign a task to an agent (async, returns immediately)
  async assign(taskRef: string, agentId: string, prompt: string): Promise<RunId> {
    const checkout = await this.checkoutTask(taskRef, agentId);
    // Spawn agent in background, return run ID
    // Agent runs, completes task, releases checkout
    return this.spawnBackground(checkout, prompt);
  }

  // Parallel: run multiple agents on same query, aggregate
  async council(input: AgentInput, agentIds: string[]): Promise<AgentOutput[]> {
    return Promise.all(agentIds.map(id => this.getAgent(id).process(input)));
  }
}
```

### Two execution modes

**Chat mode (synchronous):**
- User sends message → orchestrator routes → agent streams response → user sees it live
- Agent can call `ask_agent` tool mid-conversation for cross-domain data
- Response in seconds

**Batch mode (asynchronous):**
- Schedule or user assigns task → orchestrator creates checkout → agent runs in background
- Agent can take minutes/hours. Session persists across runs.
- Result stored in DB, notification sent to configured channels
- Used for: overnight PR generation, report building, deep analysis

### Intent routing

Simple approach first (no ML):

1. Check if message explicitly mentions an agent: "@cto what's the sprint status?"
2. Check if conversation already has an active agent — continue with same agent
3. Check if message matches agent's domain keywords (configured per agent in DB)
4. If ambiguous, use default agent or ask user

### Agent-to-agent communication

**Synchronous (via MCP tool):**

```typescript
// MCP tool available to all agents
tool('ask_agent', 'Ask another agent a question and get their response.', {
  agentId: z.string().describe('ID of the agent to ask'),
  question: z.string().describe('What to ask'),
}, async (args) => {
  const targetAgent = await loadAgent(args.agentId);
  const result = await targetAgent.process({
    text: args.question,
    source: 'agent',
    callerAgentId: currentAgent.id,
  });
  return { content: [{ type: 'text', text: result.text }] };
});
```

**Asynchronous (via task delegation):**

```typescript
// Agent creates a subtask assigned to another agent
tool('delegate_task', 'Create a subtask and assign to another agent.', {
```

### Task checkout (concurrency control)

Uses the `task_checkouts` table from 037:

```typescript
async checkoutTask(taskRef: string, agentId: string): Promise<Checkout> {
  try {
    // Atomic insert — unique index on active checkouts prevents double-work
    return db.insert(taskCheckouts).values({
      id: generateId(),
      taskRef,
      agentId,
      runId: generateRunId(),
      status: 'active',
    });
  } catch (err) {
    if (isUniqueConstraintError(err)) {
      throw new TaskAlreadyCheckedOutError(taskRef);
    }
    throw err;
  }
}

async releaseCheckout(runId: string, status: 'completed' | 'cancelled') {
  db.update(taskCheckouts)
    .set({ status, releasedAt: now() })
    .where(eq(taskCheckouts.runId, runId));
}
```

### Session persistence

Claude sessions persist across batch runs:

```typescript
// On task start: check if there's an existing session for this task
const existingSession = db.query.taskCheckouts.findFirst({
  where: and(
    eq(taskCheckouts.taskRef, taskRef),
    eq(taskCheckouts.agentId, agentId),
    isNotNull(taskCheckouts.sessionId),
  ),
  orderBy: desc(taskCheckouts.checkedOutAt),
});

// If found, resume the session
const options = existingSession?.sessionId
  ? { resumeSessionId: existingSession.sessionId }
  : {};
```

### Budget enforcement

Before each agent execution:

```typescript
async checkBudget(agentId: string): Promise<'ok' | 'soft_warning' | 'hard_stop'> {
  const policy = await getBudgetPolicy(agentId);
  if (!policy) return 'ok';

  const spent = await getMonthlySpend(agentId);
  const pct = (spent / policy.monthlyLimitCents) * 100;

  if (pct >= policy.hardThresholdPct) {
    await pauseAgent(agentId);
    await createApproval({ agentId, type: 'budget_override' });
    return 'hard_stop';
  }
  if (pct >= policy.softThresholdPct) return 'soft_warning';
  return 'ok';
}
```

### Human approval gate

Agents can request human approval for sensitive actions:

```typescript
tool('request_approval', 'Request human approval before proceeding.', {
  action: z.string().describe('What you want to do'),
  reason: z.string().describe('Why this needs human approval'),
}, async (args) => {
  const approval = await createApproval({
    agentId: currentAgent.id,
    type: 'sensitive_action',
    context: { action: args.action, reason: args.reason },
  });
  // Notify human via configured channels
  await notify(`Agent "${currentAgent.name}" requests approval: ${args.action}`);
  return {
    content: [{ type: 'text', text: `Approval requested (${approval.id}). Waiting for human decision.` }],
  };
});
```

### Conversation context

- Each conversation tracks which agent(s) participated
- Switching agents mid-conversation carries summary forward (not full context — too expensive)
- Sub-agent responses are attributed: "[CTO Agent] Cloud costs are trending up 12%..."
- Agent council responses show each agent's perspective separately
