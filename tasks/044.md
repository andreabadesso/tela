# 044 — Audit Log + Settings

**Status**: pending
**Phase**: 5 — Platform
**Dependencies**: 037, 038
**Estimated effort**: small (1-4h)

## Context

Every agent action should be logged and visible. Plus global settings need a page instead of env vars.

## Deliverables

### Audit Log page (`web/src/pages/AuditLog.tsx`)

- Filterable table of all agent actions:
  - Timestamp
  - Agent name
  - Action type (tool_call, mcp_request, knowledge_read, knowledge_write, schedule_run)
  - Details (expandable: tool name, args, result summary)
  - Source (telegram, web, cron)
  - Duration
- Filters: date range, agent, action type, source
- Export as CSV

### Audit logging middleware

Hook into agent processing to log every:
- Tool call (name, args, result length, duration)
- MCP server request (server, method, duration)
- Knowledge read/write (source, path)
- Schedule execution (schedule name, result length)

### Settings page (`web/src/pages/Settings.tsx`)

- **General:** company name, timezone, default model
- **Notification channels:** configure Telegram (bot token, chat ID), future: Slack webhook, email SMTP
- **Security:** change API token, view/rotate encryption key
- **About:** version, uptime, connected services count

## Acceptance Criteria

- [ ] Audit log page shows all agent actions
- [ ] Filters work: date, agent, action type, source
- [ ] Export to CSV functional
- [ ] Every tool call, MCP request, and knowledge operation logged automatically
- [ ] Settings page with general config
- [ ] Notification channel configuration via UI
- [ ] Settings saved to DB (not env vars)
