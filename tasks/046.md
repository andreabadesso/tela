# 046 — Docker Production Build + First Deploy

**Status**: pending
**Phase**: 5 — Platform
**Dependencies**: 036, 037, 038

## Context

Need a clean Docker build that serves both backend (API + Telegram + cron) and frontend (React SPA) in a single container. Ready to deploy on a VPS.

## Deliverables

### Multi-stage Dockerfile

```dockerfile
# Stage 1: Build frontend
FROM node:22-slim AS frontend
WORKDIR /app/web
COPY web/package*.json ./
RUN npm ci
COPY web/ ./
RUN npm run build

# Stage 2: Build backend
FROM node:22-slim AS backend
WORKDIR /app
COPY package*.json ./
RUN npm ci
COPY tsconfig.json ./
COPY src/ ./src/
RUN npm run build

# Stage 3: Runtime
FROM node:22-slim
RUN apt-get update && apt-get install -y git ripgrep && rm -rf /var/lib/apt/lists/*
WORKDIR /app
COPY --from=backend /app/dist ./dist
COPY --from=backend /app/node_modules ./node_modules
COPY --from=frontend /app/web/dist ./web/dist
COPY package.json ./
ENV NODE_ENV=production
CMD ["node", "dist/index.js"]
```

Backend serves `web/dist/` as static files at `/`. API at `/api/`. SPA fallback for client-side routing.

### Docker Compose (production)

```yaml
services:
  agent:
    build: .
    ports:
      - "3000:3000"
    env_file: .env
    volumes:
      - agent-data:/app/data    # SQLite + vector store
      - vault-data:/data/vault  # if using local vault
      - ssh-keys:/root/.ssh     # git sync
    restart: unless-stopped

  # Optional: ChromaDB for vector search
  chromadb:
    image: chromadb/chroma:latest
    volumes:
      - chroma-data:/chroma/chroma
    restart: unless-stopped

volumes:
  agent-data:
  vault-data:
  ssh-keys:
  chroma-data:
```
