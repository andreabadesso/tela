# 068 — Output Token Capping & Cost Optimization

**Status**: pending
**Phase**: 7 — Agent Runtime
**Dependencies**: 060
**Estimated effort**: small (1-4h)

## Context

Currently we use the default model output limits (up to 32K-64K tokens). The cc-tips docs show that capping output to 8K tokens covers 99% of requests and saves significant cost, since the API reserves capacity for the full max_tokens even if unused.

**Source:** cc-tips `COST_TRACKING.md`

## Deliverables

### 1. Default output token cap

Set `max_tokens: 8192` on all Claude API calls by default.

### 2. Escalation retry

When a response is truncated (model hits the cap mid-sentence):

```typescript
async function queryWithEscalation(messages, options) {
  const result = await claude.query({ ...options, max_tokens: 8192 });

  if (result.stop_reason === 'max_tokens') {
    // Response was truncated — retry with higher cap
    return claude.query({ ...options, max_tokens: 32768 });
  }

  return result;
}
```

### 3. Per-agent configurable cap

Allow agents to override the default cap in their config:

```json
{
  "id": "research-agent",
  "max_output_tokens": 16384  // This agent needs longer responses
}
```

Default: 8192. Stored in agents table config JSON.

### 4. Cost tracking integration

Enhance existing `cost_events` table logging:

- Log `tokens_out_reserved` vs `tokens_out_actual` to measure waste
- Log `escalation_retries` count per session
- Track cost savings: `(reserved_default - reserved_capped) * cost_per_token`

### 5. Cache strategy header locking

Once a cache strategy is established for a session, lock it:
- Don't change caching parameters mid-session (would bust the cache)
- Reset on new session/thread

## Acceptance Criteria

- [ ] Default max_tokens set to 8192 on all API calls
- [ ] Truncated responses automatically retry with 32K cap
- [ ] Per-agent configurable output cap
- [ ] Cost savings measurable via cost_events logging
- [ ] No UX regression: responses don't feel cut off
- [ ] Escalation retries logged for monitoring
