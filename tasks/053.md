# 053 — User-Delegated Connections

**Status**: done
**Notes**: Core OAuth flow and UI implemented; end-to-end flow may not be complete for all provider types
**Phase**: 6 — Enterprise
**Dependencies**: 050
**Estimated effort**: medium (2-8h)

## Context

Some connections should use company-wide tokens (admin connects Jira once). Others need each user to connect their own account (e.g., personal GitHub token). This task adds per-user OAuth flows.

## Deliverables

### Connection token strategies

The `connections.token_strategy` column (from 052) supports:
- `company` — admin manages one token, shared by all users
- `delegated` — each user connects their own account

### User OAuth flow

```
User clicks "Connect my GitHub" on My Connections page
  → POST /api/connections/:connectionId/auth/user
  → Backend builds OAuth URL with user's session context
  → Redirect to provider
  → Provider callback → /api/connections/:connectionId/callback/user
  → Store token in user_connections (encrypted)
  → Popup closes, UI refreshes
```

### API routes

```
GET  /api/me/connections                          — list user's connections status
POST /api/connections/:connectionId/auth/user     — initiate user OAuth
GET  /api/connections/:connectionId/callback/user — user OAuth callback
POST /api/connections/:connectionId/test/user     — test user's connection
DELETE /api/me/connections/:connectionId          — disconnect user's connection
```

### My Connections page (`web/src/pages/MyConnections.tsx`)

- For each company connection with `token_strategy = 'delegated'`:
  - Show connection name, provider icon, status (connected/not connected)
  - "Connect" button → OAuth flow
  - "Disconnect" button
  - "Test" button
- For `company` strategy connections: show as "Managed by admin" (no action needed)

### Gateway integration

The MCP Gateway (050) already resolves credentials based on `token_strategy`. This task implements the user-connection storage and OAuth that feeds into it.

### Navigation

Add "My Connections" to the main sidebar (not admin — visible to all users).
