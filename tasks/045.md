# 045 — Notification Channels (Pluggable)

**Status**: pending
**Phase**: 5 — Platform
**Dependencies**: 037, 043, 044
**Estimated effort**: medium (2-8h)

## Context

Currently Telegram only. Schedules and alerts need to go to configurable channels: Telegram, Slack, web UI, email, webhook.

## Deliverables

### NotificationChannel interface

```typescript
interface NotificationChannel {
  id: string;
  type: 'telegram' | 'slack' | 'web' | 'email' | 'webhook';
  name: string;
  config: Record<string, string>; // type-specific config

  send(message: NotificationMessage): Promise<void>;
  test(): Promise<boolean>;
}

interface NotificationMessage {
  title?: string;
  body: string;
  priority: 'low' | 'normal' | 'high';
  source: string; // which schedule or agent sent this
}
```

### Channels to implement

| Channel | Config needed | Implementation |
|---|---|---|
| **Telegram** | bot_token, chat_id | Already exists, refactor into interface |
| **Web UI** | none | WebSocket push to frontend, in-app notification bell |
| **Slack** | webhook_url | Slack Incoming Webhook |
| **Email** | smtp_host, smtp_port, from, to | Nodemailer |
| **Webhook** | url, headers | HTTP POST with JSON body |

### UI integration

- Settings page: add/configure/test/remove notification channels
- Schedules page: select which channels each schedule notifies
- Agent config: default notification channel for that agent

### Web UI notifications

- Notification bell icon in top nav
- Badge count for unread
- Dropdown with recent notifications
- Click to expand full output
- Mark as read
