# 047 — Auth & Identity Database Schema

**Status**: pending
**Phase**: 6 — Enterprise
**Dependencies**: 037
**Estimated effort**: medium (2-8h)

## Context

Tela is going from single-user to 140-person enterprise deployment. This task lays the database foundation for users, roles, teams, and access policies. Everything else in Phase 6 builds on these tables.

## Deliverables

### New migration: `004_auth_rbac.sql`

```sql
-- Users (extended by better-auth)
CREATE TABLE IF NOT EXISTS users (
  id TEXT PRIMARY KEY,
  email TEXT NOT NULL UNIQUE,
  name TEXT,
  avatar_url TEXT,
  status TEXT NOT NULL DEFAULT 'active',  -- active, suspended, deactivated
  created_at TEXT NOT NULL DEFAULT (datetime('now')),
  updated_at TEXT NOT NULL DEFAULT (datetime('now'))
);

-- Sessions (better-auth schema)
CREATE TABLE IF NOT EXISTS sessions (
  id TEXT PRIMARY KEY,
  user_id TEXT NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  token TEXT NOT NULL UNIQUE,
  expires_at TEXT NOT NULL,
  created_at TEXT NOT NULL DEFAULT (datetime('now'))
);

-- Accounts (better-auth OAuth accounts)
CREATE TABLE IF NOT EXISTS accounts (
  id TEXT PRIMARY KEY,
  user_id TEXT NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  provider TEXT NOT NULL,
  provider_account_id TEXT NOT NULL,
  access_token TEXT,
  refresh_token TEXT,
  expires_at TEXT,
  created_at TEXT NOT NULL DEFAULT (datetime('now')),
  UNIQUE(provider, provider_account_id)
);

-- API Keys for programmatic access
CREATE TABLE IF NOT EXISTS user_api_keys (
  id TEXT PRIMARY KEY,
  user_id TEXT NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  name TEXT NOT NULL,
  key_hash TEXT NOT NULL UNIQUE,
  key_prefix TEXT NOT NULL,
  scopes TEXT NOT NULL DEFAULT '[]',
  last_used_at TEXT,
  expires_at TEXT,
  created_at TEXT NOT NULL DEFAULT (datetime('now'))
);

-- Roles
CREATE TABLE IF NOT EXISTS roles (
  id TEXT PRIMARY KEY,
  name TEXT NOT NULL UNIQUE,
  description TEXT,
  is_system INTEGER NOT NULL DEFAULT 0,
  created_at TEXT NOT NULL DEFAULT (datetime('now'))
);

-- Teams (departments, squads)
CREATE TABLE IF NOT EXISTS teams (
  id TEXT PRIMARY KEY,
  name TEXT NOT NULL UNIQUE,
  description TEXT,
  created_at TEXT NOT NULL DEFAULT (datetime('now'))
);

-- User <-> Role
CREATE TABLE IF NOT EXISTS user_roles (
  user_id TEXT NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  role_id TEXT NOT NULL REFERENCES roles(id) ON DELETE CASCADE,
  PRIMARY KEY (user_id, role_id)
);

-- User <-> Team
CREATE TABLE IF NOT EXISTS user_teams (
  user_id TEXT NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  team_id TEXT NOT NULL REFERENCES teams(id) ON DELETE CASCADE,
  role_in_team TEXT NOT NULL DEFAULT 'member',
  PRIMARY KEY (user_id, team_id)
);
```

### Seed data

Default roles (is_system = 1, cannot be deleted):

| ID | Name | Description |
|----|------|-------------|
| admin | Admin | Full platform access. Manages users, connections, agents, policies. |
| engineering | Engineering | Access to GitHub, Jira, CI/CD, monitoring, ShipLens. Engineering docs. |
| finance | Finance | Access to financial systems, budget tools. Financial docs. |
| sales | Sales | Access to CRM, pipeline tools. Sales docs. |
| hr | HR | Access to people tools, payroll (read). HR docs. |
| leadership | Leadership | Read access to all connections and knowledge. All agents. |
| viewer | Viewer | No MCP access. Default agent only. For trainees and new hires. |
