# 051 — MCP Governance Policies Admin UI

**Status**: done
**Phase**: 6 — Enterprise
**Dependencies**: 049, 050
**Estimated effort**: medium (2-8h)

## Context

Admins need a visual interface to manage who can access what. This is the policy management UI for the MCP Gateway.

## Deliverables

### Policies page (`web/src/pages/admin/Policies.tsx`)

Tabbed interface with three sections:

#### MCP Policies tab

- Table: Principal (role/team/user icon + name) | Connection | Access Level | Allowed Tools | Rate Limit
- "Add Policy" button → dialog:
  - Principal type selector: Role / Team / User
  - Principal selector (dropdown of roles/teams/users based on type)
  - Connection selector (dropdown of all connections)
  - Access level: None / Read / Write / Admin
  - Allowed tools: multi-select or "all" (auto-populated from MCP server tool discovery)
  - Denied tools: multi-select
  - Max data classification: Public / Internal / Confidential / Restricted
  - Rate limit per hour (optional)
  - Rate limit per day (optional)
- Edit and delete on each row
- Bulk actions: "Apply template" → select a role template and auto-create policies

#### Knowledge Policies tab

- Table: Principal | Knowledge Source | Access Level
- Same CRUD pattern

#### Agent Policies tab

- Table: Principal | Agent | Can Use | Can Configure
- Same CRUD pattern

### Tool classification page (`web/src/pages/admin/ToolClassifications.tsx`)

- Per-connection: list all discovered tools with their classification
- Dropdown per tool: Public / Internal / Confidential / Restricted
- Auto-save on change
- "Classify all as Internal" bulk action

### Navigation

Add "Admin" section to sidebar (only visible to admin role):
- Users
- Roles
- Teams
- Policies
- Tool Classifications

### Access matrix view

Read-only summary view showing a matrix:
- Rows: roles
- Columns: connections
- Cells: access level (color-coded: green=write, yellow=read, red=none)

Gives admin an at-a-glance view of "who can access what."

## Acceptance Criteria
