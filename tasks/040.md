## Context

Credentials are in .env. Adding a new integration means editing files and restarting. This task builds a UI-driven connection manager where you click "Connect Jira" → OAuth flow → done.

## Deliverables

### Connections page (`web/src/pages/Connections.tsx`)

- Grid of available integrations (cards):
  - Jira (Atlassian OAuth 2.0)
  - GitHub (GitHub OAuth App)
  - Google Calendar + Gmail (Google OAuth 2.0)
  - Slack (Slack OAuth)
  - Notion (Notion OAuth)
  - ShipLens (API key)
  - Custom MCP Server (URL + optional auth)
- Each card shows: icon, name, status (connected/disconnected/error), last sync time
- "Connect" button → initiates OAuth flow in popup/redirect
- "Disconnect" button → revokes tokens, removes from DB
- "Test" button → pings the service to verify connection

### OAuth flow

```
User clicks "Connect Jira"
  → Frontend opens /api/connections/jira/auth (new window)
  → Backend redirects to Atlassian OAuth consent screen
  → User authorizes
  → Atlassian redirects to /api/connections/jira/callback
  → Backend stores tokens (encrypted) in connections table
  → Callback page closes, frontend refreshes status
```

### Backend

```
GET  /api/connections                    — list all connections with status
POST /api/connections                    — create connection config
GET  /api/connections/:type/auth         — initiate OAuth flow
GET  /api/connections/:type/callback     — OAuth callback handler
POST /api/connections/:type/test         — test connectivity
DELETE /api/connections/:id              — disconnect + revoke tokens
```

### Token management

- Tokens encrypted with AES-256-GCM using `ENCRYPTION_KEY` env var
- Refresh tokens auto-refreshed before expiry
- Connection status updated on each use (connected → error if fails)
- Token refresh failures trigger notification

### MCP Server Registry

Each connected service registers as an MCP server available to agents:

```typescript
// When Jira connects, register MCP server
mcpRegistry.register({
  id: 'jira',
  name: 'Jira',
  connectionId: connection.id,
  // Uses Atlassian's official Remote MCP Server
  url: 'https://mcp.atlassian.com/v1',
  auth: () => getDecryptedToken(connection.id),
});
```

Agents configured with `mcp_servers: ["jira"]` automatically get access.
