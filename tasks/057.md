# 057 — MCP Server Registry (Dynamic Discovery)

**Status**: pending
**Phase**: 6 — Enterprise
**Dependencies**: 050
**Estimated effort**: medium (2-8h)

## Context

Today MCP servers are configured via env vars (ShipLens URL) or hardcoded (vault tools). For enterprise deployment, admins need to add MCP servers from the UI and have them automatically available to agents (governed by policies).

## Deliverables

### MCP Server Registry service (`src/services/mcp-registry.ts`)

```typescript
export class McpServerRegistry {
  private clients = new Map<string, McpClient>();

  // Create an MCP client for a connection
  async connect(connection: ConnectionRow): Promise<void>;

  // Disconnect and clean up
  async disconnect(connectionId: string): Promise<void>;

  // Get available tools for a connection
  async discoverTools(connectionId: string): Promise<ToolDefinition[]>;

  // Health check
  async healthCheck(connectionId: string): Promise<boolean>;

  // Get MCP client for proxying tool calls
  getClient(connectionId: string): McpClient | undefined;
}
```

### Transport support

Based on `connections.type` and config:

| Connection type | Transport | How it connects |
|----------------|-----------|----------------|
| ShipLens | SSE / Streamable HTTP | `url` in config |
| Custom MCP | Streamable HTTP | `mcp_server_url` |
| Remote MCP (Atlassian, etc.) | Streamable HTTP | Official MCP endpoint URLs |
| Local MCP | stdio | `command` in config |

### Auto-discovery

When a connection status changes to 'connected':
1. Registry connects to the MCP server
2. Discovers all available tools (via MCP `tools/list`)
3. Stores tool names + descriptions in a cache
4. Admin can classify tools (via 052's `mcp_tool_classifications`)

### Health monitoring

Background job (every 5 minutes):
- Ping each registered MCP server
- Update `connections.status` on failure
- Reconnect on recovery

### Wire into MCP Gateway

The Gateway (050) uses the registry to get actual MCP clients for proxying:

```typescript
// In McpGateway.createGovernedProxy():
const client = this.registry.getClient(connectionId);
// Proxy tool calls through the client
```

### Admin UI enhancement

On the Connections page:
- Show discovered tools per connection
- Tool count badge on each connection card
- "Refresh Tools" button to re-discover
