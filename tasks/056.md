# 056 — Enterprise Hardening

**Status**: pending
**Phase**: 6 — Enterprise
**Dependencies**: 048, 050, 054, 055
**Estimated effort**: large (8-20h)

## Context

Prepare Tela for 140 concurrent users in production. Security, performance, reliability.

## Deliverables

### Session management

- Concurrent session limit (e.g., 3 active sessions per user)
- Session revocation by admin ("force sign out")
- Session activity tracking (last active, IP, user-agent)
- Auto-expire inactive sessions (7 days)

### SQLite → PostgreSQL evaluation

140 users = many concurrent writes (conversations, audit, cost events). Evaluate:
- Run load test: 30 concurrent users, 2 messages/minute each
- Measure SQLite WAL write latency (p50, p95, p99)
- If p99 > 100ms → implement Postgres adapter
- Migration path: same SQL (ANSI-compatible), swap better-sqlite3 for pg

### Security hardening

- CSRF protection on all mutation endpoints
- Input validation (zod) on all API routes
- SQL injection prevention audit (parameterized queries — already done, but verify)
- XSS prevention in frontend (React handles this, but verify raw HTML rendering)
- Rate limiting on auth endpoints (brute force protection)
- Helmet-style security headers via Hono middleware

### Error handling

- Global error handler that never leaks stack traces to users
- Structured error responses: `{ error: string, code: string }`
- Agent errors don't crash the server (already mostly handled, audit)

### Monitoring

- Prometheus-compatible metrics endpoint `/api/metrics`:
  - Active users (sessions)
  - Requests per second
  - Agent query duration histogram
  - MCP tool call count by connection
  - Error rate
  - Cost accumulator
- Health endpoint enhanced with user counts

### Data export

- Admin can export audit log as CSV
- Admin can export user list as CSV
- GDPR-style user data export ("download my data")

### Backup strategy

- SQLite: document WAL-safe backup procedure
- Postgres: pg_dump cron recommendation

## Acceptance Criteria

- [ ] Concurrent session limits enforced
- [ ] Admin can force sign out any user
- [ ] Load test: 30 concurrent users sustained without errors
- [ ] SQLite vs Postgres decision documented with benchmark results
- [ ] All API routes have input validation
- [ ] Security headers present
- [ ] Auth endpoints rate limited
- [ ] `/api/metrics` endpoint for monitoring
- [ ] CSV export for audit log and user list
