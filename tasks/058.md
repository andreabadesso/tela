## Context

When Tela is first deployed for the company, the admin needs a guided setup experience. And when each of the 140 employees first logs in, they should get a smooth onboarding.

## Deliverables

### First-run setup wizard (`web/src/pages/Setup.tsx`)

Shown on first boot (no users in DB). Steps:

1. **Welcome** — "Welcome to Tela. Let's set up your company OS."
2. **Admin account** — Sign in with Google (this user becomes admin)
3. **Company info** — Company name, timezone, default model
4. **Connect tools** — Quick-connect cards for Jira, GitHub, Google, Slack, ShipLens
5. **Create teams** — Add departments (Engineering, Finance, Sales, HR, Leadership)
6. **Invite users** — Enter email domain for auto-provisioning (e.g., `@example.com` → any Google SSO from this domain auto-creates account)
7. **Set policies** — Apply role templates (Engineering role → GitHub+Jira, etc.)
8. **Done** — "Tela is ready. Share the URL with your team."

### Employee onboarding flow

When a new user logs in for the first time:

1. **Welcome screen** — "Hi {name}! Welcome to Tela."
2. **Your role** — Show assigned role and what tools are available
3. **Connect personal accounts** (if any delegated connections) — "Connect your GitHub to get personalized results"
4. **Quick tour** — Highlight Chat, available agents, key features for their role
5. **Start chatting** — Redirect to Chat with a suggested first message

### Domain-based auto-provisioning

Setting: `allowed_email_domains` (e.g., `["example.com"]`)
- Any Google SSO login from an allowed domain → auto-create user with `viewer` role
- Logins from other domains → rejected
- Admin can override and invite specific external emails

### API routes

```
GET  /api/setup/status     — check if first-run setup is needed
POST /api/setup/complete   — mark setup as complete
GET  /api/onboarding       — get onboarding status for current user
POST /api/onboarding/complete — mark user onboarding as complete
```

### Database

```sql

```

Add setting: `setup_completed` (boolean in settings table).
