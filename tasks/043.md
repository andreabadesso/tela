# 043 — Schedules UI (Visual Cron)

**Status**: pending
**Phase**: 5 — Platform
**Dependencies**: 037, 038, 039
**Estimated effort**: medium (2-8h)

## Context

12 hardcoded cron jobs in `src/jobs/`. This task moves them to DB-defined schedules configurable from the UI. The existing jobs become templates.

## Deliverables

### Schedules page (`web/src/pages/Schedules.tsx`)

- List all schedules: name, cron expression (human-readable), agent, enabled/disabled, last run, last result (truncated)
- "Create Schedule" → form: name, cron expression (with helper picker), select agent, prompt textarea, notification channels (multi-select), enabled toggle
- Edit existing schedule
- Delete schedule
- Toggle enabled/disabled
- "Run Now" button → executes immediately, shows result
- History: expand a schedule to see last 10 runs with output

### Templates

Convert existing 12 cron jobs into templates:

| Template | Default Cron | Default Prompt |
|---|---|---|
| Morning Briefing | `0 8 * * *` | "Generate morning briefing with priorities, attention items, meetings, carry-over" |
| Midday Check | `0 12 * * *` | "Quick midday status: anything urgent or blocked?" |
| Daily Digest | `0 20 * * *` | "End of day digest: what happened, what's pending" |
| End of Day | `0 21 * * *` | "Prompt for end of day reflection and review" |
| Week Ahead | `0 20 * 0` | "Plan next week based on calendar, roadmap, pending items" |
| Week Review | `0 17 * 5` | "Weekly review: what shipped, metrics, highlights, concerns" |
| Meeting Prep | `*/15 * * * *` | "Check for meetings in next 30min and prep notes" |
| PR Stale Alerts | `0 10 * * *` | "Check for PRs open more than 48h without review" |
| Blocked Tickets | `0 10 * * *` | "Check for Jira tickets blocked more than 24h" |
| ShipLens Anomaly | `0 */4 * * *` | "Check ShipLens for metric anomalies" |
| Decision Review | `0 9 * * 1` | "Review decisions from last 30 days: outcomes match expectations?" |
| Vault Health | `0 9 * * 6` | "Run vault health check: orphans, broken links, stale items" |

User creates from template → edits cron/prompt/agent → saves.

### Backend

- `JobRegistry` reads schedules from DB instead of hardcoded registrations
- On schedule create/update/delete: dynamically add/remove/update node-cron jobs
- Schedule execution: load agent config → run `agent.process(prompt)` → store result → send notification
- Notification channels: Telegram, Web UI (in-app notification), future: Slack, email

### Remove hardcoded jobs

Delete all files in `src/jobs/` except `registry.ts`. Registry reads from DB.
