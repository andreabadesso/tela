## Context

First frontend for Tela. Uses [assistant-ui](https://www.assistant-ui.com/) for the chat interface — headless React primitives for AI chat with built-in streaming, tool call rendering, thread management, and MCP support. Shadcn/ui for the rest of the app (layout, forms, tables, etc.).

assistant-ui is the standard React library for agentic chat UIs. No reason to build message rendering, streaming indicators, or tool call cards from scratch.

### assistant-ui integration

Use `useExternalStoreRuntime` adapter to connect assistant-ui to Tela's backend API/WebSocket:

```typescript
// lib/tela-runtime.ts
import { useExternalStoreRuntime } from "@assistant-ui/react";

// Connects assistant-ui's thread/message model
// to Tela's /api/chat/stream WebSocket endpoint
export function useTelaRuntime(agentId: string) {
  return useExternalStoreRuntime({
    messages,
    isRunning,
    onNew: async (message) => {
      // Send to Tela API via WebSocket
      // Receive streaming chunks: thinking, tool_call, text, done
      // assistant-ui handles rendering automatically
    },
  });
}
```

### Chat page using assistant-ui primitives

```tsx
import { Thread, ThreadMessages, Composer } from "@assistant-ui/react";

export function Chat() {
  const runtime = useTelaRuntime(selectedAgentId);

  return (
    <AssistantRuntimeProvider runtime={runtime}>
      <Thread>
        <ThreadMessages />
        <Composer />
      </Thread>
    </AssistantRuntimeProvider>
  );
}
```

assistant-ui handles out of the box:
- Message streaming with typing indicator
- Tool call rendering (expandable cards showing name, args, result)
- Message branching (edit and regenerate)
- Thread management (conversation list)
- Attachment handling
- Markdown rendering in messages

### What we customize on top of assistant-ui

- **Agent selector dropdown** in the composer area (which agent to talk to)
- **Tool call card styling** to match Tela's design language
- **Agent attribution badge** showing which agent responded (for multi-agent conversations)
- **Approval inline UI** — when agent requests approval, show approve/reject buttons in the chat
