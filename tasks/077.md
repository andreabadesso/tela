# Task 077 — ProjectChat UX fixes

## Bug 1: Preview shows JSON error before agent serves anything

**Symptom**: iframe shows `{"error":"no_port","message":"No exposed port available for this application"}` as soon as the project page loads, before the agent calls `serve_workspace_app`.

**Cause**: `project.app_url` is set from workspace ID at creation time. The app proxy has nothing to serve yet.

**Fix**: In `web/src/pages/ProjectChat.tsx`, before rendering the `<iframe>`, check if the workspace actually has something served:
- Fetch `GET /api/workspaces/:workspaceId` and check `static_app_dir` or `port_mappings`
- If both are empty AND a session is active → show a placeholder ("Agent is building...") instead of the iframe
- Switch to iframe once `static_app_dir` is set OR `port_mappings` is non-empty

Alternative: simpler — just show placeholder while `isRunning && !hasServedAnything`, where `hasServedAnything` is tracked by listening to the SSE stream for a `tool_result` from `serve_workspace_app`.

## Bug 2: Chat history lost on page refresh

**Scenario A — Active session**: `streamOverrides` is React state, cleared on refresh. Fix:
- In `useProjectChatRuntime`, persist stream state to `localStorage` keyed by `sessionId` as events arrive
- On init, seed `streamOverrides` from `localStorage` for any session with `status === 'running'`
- Clear localStorage entry when session transitions to committed/failed

**Scenario B — Committed sessions**: Should load from DB via TanStack Query. Investigate if this is also affected (race condition, empty state flash, or scroll issue).
