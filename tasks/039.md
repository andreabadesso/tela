# 039 — Agent Configuration UI

**Status**: pending
**Phase**: 5 — Platform
**Dependencies**: 037, 038
**Estimated effort**: medium (2-8h)

## Context

Agents are currently hardcoded in agent.ts. This task makes them configurable from the web UI — create, edit, toggle, delete. Each agent has: name, model, system prompt, connected MCP servers, knowledge sources, permissions.

## Deliverables

### Agents page (`web/src/pages/Agents.tsx`)

- List all agents with status (enabled/disabled), model, last active
- Create new agent (form): name, model dropdown, system prompt textarea, MCP server multi-select, knowledge source multi-select, permission toggles
- Edit existing agent
- Delete agent (with confirmation)
- Toggle enabled/disabled
- "Duplicate" button (copy an agent to create a variant)

### Agent Templates

Pre-built templates user can start from:

- **CTO Agent** — engineering focus, connects to Jira/GitHub/CI/Monitoring
- **CEO Agent** — full access, strategic focus
- **CFO Agent** — finance focus
- **Support Agent** — customer support focus
- **Blank** — empty, user builds from scratch

Templates are just prefilled form values, not special code.

### Backend changes

- Refactor `CtoAgent` class to be generic `Agent` class instantiated from DB config
- `agent.process()` reads agent config from DB at invocation time (not constructor)
- Multiple agents can coexist — orchestrator routes to the right one
- System prompt supports `{{variables}}`: `{{company_name}}`, `{{today}}`, `{{agent_name}}`

## Acceptance Criteria

- [ ] Agents page lists all agents with status
- [ ] Create agent from blank or template
- [ ] Edit agent: all fields editable
- [ ] Delete with confirmation
- [ ] Enable/disable toggle
- [ ] Duplicate button
