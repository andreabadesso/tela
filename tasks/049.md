## Context

With users authenticated, we need to enforce what each user can access. This task builds the permission resolution engine that all other governance features depend on.

## Deliverables

### Permission resolution service (`src/services/rbac.ts`)

```typescript
export interface EffectivePermissions {
  // MCP connections this user can access, with level
  mcpAccess: Map<string, {
    connectionId: string;
    accessLevel: 'none' | 'read' | 'write' | 'admin';
    allowedTools: string[] | null;   // null = all allowed
    deniedTools: string[] | null;    // null = none denied
    maxDataClassification: string;
    rateLimitPerHour: number | null;
    rateLimitPerDay: number | null;
  }>;
  // Knowledge sources
  knowledgeAccess: Map<string, 'none' | 'read' | 'write'>;
  // Agents
  agentAccess: Map<string, { canUse: boolean; canConfigure: boolean }>;
  // Platform permissions
  platformPermissions: Set<string>;
}

export class RbacService {
  constructor(private db: DatabaseService) {}

  // Main method: resolve all permissions for a user
  getEffectivePermissions(userId: string): EffectivePermissions;

  // Convenience checks
  canAccessMcp(userId: string, connectionId: string): McpAccessLevel;
  canUseAgent(userId: string, agentId: string): boolean;
  canAccessKnowledge(userId: string, sourceId: string): 'none' | 'read' | 'write';
  isAdmin(userId: string): boolean;
  hasPermission(userId: string, permission: string): boolean;
}
```

### Permission resolution algorithm

```
For each resource (connection/knowledge/agent):
  1. Collect all policies where principal matches:
     - user_id directly
     - any of user's roles
     - any of user's teams
  2. If ANY policy says 'none' → deny (deny override)
  3. Otherwise, take MOST PERMISSIVE across all policies
  4. If no policies exist → deny (default deny)
```

### Route protection middleware

```typescript
function requireRole(...roles: string[]) {
  return async (c: Context, next: Next) => {
    const user = c.get('user');
    if (!user) return c.json({ error: 'Unauthorized' }, 401);
    if (!roles.some(r => user.roles.includes(r))) {
      return c.json({ error: 'Forbidden' }, 403);
    }
    return next();
  };
}
```

Apply to existing admin routes: agent management, connection management, schedule management.
