# 048 — Authentication (better-auth + Google SSO)

**Status**: pending
**Phase**: 6 — Enterprise
**Dependencies**: 047
**Estimated effort**: large (8-20h)

## Context

Replace the single `API_TOKEN` with real user authentication. 140 employees log in via Google Workspace SSO. API keys for programmatic/CLI access.

## Deliverables

### Install better-auth

```bash
npm install better-auth
```

### Configure better-auth (`src/auth/index.ts`)

```typescript
import { betterAuth } from 'better-auth';

export const auth = betterAuth({
  database: { /* SQLite via better-sqlite3 */ },
  socialProviders: {
    google: {
      clientId: process.env.GOOGLE_SSO_CLIENT_ID!,
      clientSecret: process.env.GOOGLE_SSO_CLIENT_SECRET!,
    },
  },
  session: {
    expiresIn: 60 * 60 * 24 * 7, // 7 days
  },
});
```

Note: This is a SEPARATE Google OAuth app from the Google Calendar integration. This one is for user login. The env vars are `GOOGLE_SSO_CLIENT_ID` / `GOOGLE_SSO_CLIENT_SECRET`.

### Rewrite auth middleware (`src/api/middleware.ts`)

The current middleware checks a single `API_TOKEN`. Replace with:

1. Check for session cookie → resolve user from `sessions` table
2. Check for `Authorization: Bearer <api_key>` → resolve user from `user_api_keys` table (hash and lookup)
3. Fallback: check for legacy `API_TOKEN` env var (super-admin escape hatch during migration)
4. If no valid auth → 401

Set `c.set('user', { id, email, name, roles, teams })` on every authenticated request.

### Mount auth routes

In `src/api/server.ts`, mount better-auth handler at `/api/auth/*`. These routes handle:
- `POST /api/auth/sign-in/social` — initiate Google SSO
- `GET /api/auth/callback/google` — SSO callback
- `POST /api/auth/sign-out` — destroy session
- `GET /api/auth/session` — get current session

### Auto-provision users

On first SSO login:
1. Create `users` row from Google profile (email, name, avatar)
2. Assign `viewer` role by default
3. Admin upgrades role later

### API key management routes

```
POST   /api/auth/api-keys          — create key (returns plaintext once)
GET    /api/auth/api-keys          — list user's keys (prefix only, never hash)
DELETE /api/auth/api-keys/:id      — revoke key
```

### Login page in frontend

`web/src/pages/Login.tsx`:

### Frontend auth wrapper

`web/src/lib/auth.ts`:
- `useSession()` hook — fetches current session

### Update WebSocket auth

In `src/api/ws.ts`, validate session token instead of `API_TOKEN`. Resolve user and pass `userId` to agent processing.
