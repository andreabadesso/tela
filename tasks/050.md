## Context

This is the architectural heart of enterprise Tela. The MCP Gateway sits between agents and MCP servers, enforcing per-user permissions on every tool call.

## The Problem

Today in `AgentService.process()`, MCP servers are injected directly:
```typescript
mcpServers: { 'vault-tools': this.mcpServer, ...externalServers }
```
Every user gets the same servers. A trainee sees the same tools as the CTO.

## The Solution

The MCP Gateway creates **proxy MCP servers** that wrap real ones with an authorization layer.

## Deliverables

### MCP Gateway service (`src/services/mcp-gateway.ts`)

```typescript
export class McpGateway {
  constructor(
    private db: DatabaseService,
    private rbac: RbacService,
    private encryption: EncryptionService,
  ) {}

  /**
   * Resolve the governed MCP server map for a (user, agent) pair.
   * Returns only servers the user is permitted to access,
   * wrapped in authorization proxies.
   */
  async resolveServers(
    userId: string,
    agentId: string,
  ): Promise<Record<string, McpServerConfig>> {
    // 1. Get agent's configured MCP servers
    const agent = this.db.getAgent(agentId);
    const agentServers: string[] = JSON.parse(agent.mcp_servers);

    // 2. Get user's effective MCP permissions
    const perms = this.rbac.getEffectivePermissions(userId);

    // 3. Intersect: agent wants it AND user has access
    const allowed = agentServers.filter(id => {
      const access = perms.mcpAccess.get(id);
      return access && access.accessLevel !== 'none';
    });

    // 4. For each allowed server, create a governed proxy
    const servers: Record<string, McpServerConfig> = {};
    for (const connectionId of allowed) {
      const policy = perms.mcpAccess.get(connectionId)!;
      const connection = this.db.getConnection(connectionId);
      if (!connection || connection.status !== 'connected') continue;

      servers[connectionId] = this.createGovernedProxy(
        userId, connectionId, connection, policy,
      );
    }

    // Always include vault-tools (governed by knowledge policies)
    // ... vault proxy with knowledge access checks

    return servers;
  }
}
```

### Governed proxy MCP server

For each real MCP server, create a proxy using `createSdkMcpServer`:

```typescript
private createGovernedProxy(userId, connectionId, connection, policy) {
  // Discover tools from the real MCP server
  const realTools = this.getRealTools(connection);

  // Filter tools based on policy
  const allowedTools = realTools.filter(tool => {
    if (policy.deniedTools?.includes(tool.name)) return false;
    if (policy.allowedTools && !policy.allowedTools.includes(tool.name)) return false;
    return true;
  });

  // Create proxy tools that wrap each real tool with:
  // 1. Access level check (read vs write)
  // 2. Data classification check
  // 3. Rate limit check
  // 4. Credential injection
  // 5. Audit logging
  return createSdkMcpServer({
    name: `governed-${connectionId}`,
    tools: allowedTools.map(tool => this.wrapTool(
      tool, userId, connectionId, policy
    )),
  });
}
```

### Tool wrapping

Each tool call goes through:

```typescript
private wrapTool(realTool, userId, connectionId, policy) {
  return tool(realTool.name, realTool.description, realTool.schema, async (args) => {
    // 1. Check write permission if this is a write operation
    if (isWriteOperation(realTool.name) && policy.accessLevel === 'read') {
      return { content: [{ type: 'text', text: `Permission denied: read-only access to ${connectionId}` }] };
    }

    // 2. Check data classification
    const classification = this.db.getToolClassification(connectionId, realTool.name);
    if (classification && !isClassificationAllowed(classification, policy.maxDataClassification)) {
      return { content: [{ type: 'text', text: `Permission denied: ${realTool.name} requires ${classification} clearance` }] };
    }

    // 3. Check rate limit
    if (!this.rateLimiter.check(userId, connectionId, policy)) {
      return { content: [{ type: 'text', text: 'Rate limit exceeded. Try again later.' }] };
    }

    // 4. Log the attempt
    this.db.logAudit(null, 'mcp_tool_call', {
      tool: realTool.name,
      args: sanitizeArgs(args),
      connectionId,
    }, 'web', undefined, userId, connectionId, realTool.name, 'allowed');

    // 5. Forward to real MCP server with correct credentials
    const credentials = await this.resolveCredentials(userId, connectionId);
    return realTool.handler(args, { credentials });
  });
}
```

### Credential resolution

Two strategies:

```typescript
async resolveCredentials(userId: string, connectionId: string): Promise<string> {
  const connection = this.db.getConnection(connectionId);

  if (connection.token_strategy === 'delegated') {
    // User-specific token
    const userConn = this.db.getUserConnection(userId, connectionId);
    if (!userConn?.credentials) {
      throw new Error(`Please connect your ${connection.name} account in Settings > My Connections`);
    }
    return this.encryption.decrypt(userConn.credentials);
  }

  // Company-wide token
  return this.encryption.decrypt(connection.credentials!);
}
```
